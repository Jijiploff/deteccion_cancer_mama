1. INTRODUCCIÓN
===============================================================================

El sistema de apoyo diagnóstico para clasificación de mamografías utiliza
modelos de deep learning (EfficientNet) y machine learning (XGBoost,
Random Forest) entrenados con los datasets CBIS-DDSM y Wisconsin Breast
Cancer Database para estimar la probabilidad de que un hallazgo sea
benigno o maligno.

  IMPORTANTE: Esta herramienta es de APOYO DIAGNÓSTICO.
  No reemplaza el criterio de un profesional médico calificado.

El sistema consta de dos interfaces principales:

  A. Aplicación Web (React)   → Diagnóstico de mamografías
  B. Training Dashboard (Streamlit) → EDA, entrenamiento y reportes

3. APLICACIÓN WEB - INTERFAZ DE USUARIO
===============================================================================

3.1 Login

  Al abrir la aplicación, se muestra la pantalla de inicio de sesión.
  Ingresa tus credenciales y haz clic en "Ingresar".

  Si las credenciales son incorrectas, se mostrará un mensaje de error.

3.2 Encabezado (Header)

  Una vez autenticado, el encabezado muestra:
  - Logo "BC" (Breast Cancer)
  - Título "Apoyo Diagnóstico - Clasificación de mamografías"
  - Nombre de usuario
  - Botón "Dashboard" (acceso al panel)
  - Botón "Salir" (cerrar sesión)
  - Botón de cambio de modo claro/oscuro

3.3 Sección Hero

  Descripción del sistema y aviso sobre que es una herramienta de apoyo.

3.4 Panel de Carga (UploadPanel)

  - Área de arrastrar y soltar (drag & drop)
  - Haz clic para seleccionar archivo desde el explorador
  - Formatos soportados: JPG y PNG
  - Tamaño máximo: 5MB
  - Al seleccionar una imagen, se muestra una vista previa
    y comienza automáticamente el análisis

3.5 Formulario Clínico (ClinicalForm)

  Sección expandible para ingresar datos clínicos adicionales:
  - BI-RADS Assessment (2-5): categoría de evaluación
  - Subtlety (1-5): grado de sutileza del hallazgo
  - Edad del paciente
  - Densidad mamaria (1-4)
  - 30 características del núcleo celular (Wisconsin)

  Los valores de Wisconsin se pueden dejar por defecto (medias del dataset).

3.6 Panel de Resultados (ResultPanel)

  Una vez procesada la imagen, el panel muestra:

  - Label: BENIGN (benigno) en verde / MALIGNANT (maligno) en rojo
  - Confianza: porcentaje de confianza en la predicción
  - Barra de probabilidad: visualización comparativa benigno vs maligno
  - Advertencia: si la confianza es baja (<65%)

  Cada modelo (CNN, Ensemble, XGBoost) muestra su resultado individual
  en una tabla expandible.

3.7 Historial (HistoryList)

  - Muestra los últimos 50 diagnósticos realizados
  - Cada entrada: nombre de archivo, resultado (BENIGN/MALIGNANT),
    confianza y fecha/hora
  - Botón "Limpiar" para borrar todo el historial
  - Se actualiza automáticamente después de cada diagnóstico

3.8 Dashboard

  Accesible desde el botón "Dashboard" en el encabezado.

  Muestra:
  - Total de diagnósticos realizados
  - Cantidad de casos malignos y benignos
  - Estado de la API (online/offline)
  - Tabla del historial completo con los últimos 20 diagnósticos

4. APLICACIÓN WEB - FLUJO DE USO
===============================================================================

4.1 Realizar un diagnóstico

  Paso 1: Abrir la aplicación e iniciar sesión
  Paso 2: Arrastrar una mamografía (JPG/PNG) al panel de carga
  Paso 3: (Opcional) Expandir el formulario clínico para ingresar
           datos adicionales
  Paso 4: Esperar el procesamiento (unos segundos)
  Paso 5: Revisar el resultado:
           - BENIGN (verde) → probablemente benigno
           - MALIGNANT (rojo) → probablemente maligno
  Paso 6: Verificar la confianza. Si es <65%, el sistema muestra
           una advertencia: se recomienda revisión adicional
  Paso 7: El resultado se agrega automáticamente al historial

4.2 Interpretar resultados

  BENIGN (Confianza > 90%):   Alta probabilidad de hallazgo benigno
  BENIGN (Confianza 65-90%):  Probabilidad de hallazgo benigno
  MALIGNANT (Confianza > 90%): Alta probabilidad de hallazgo maligno
  MALIGNANT (Confianza 65-90%): Probabilidad de hallazgo maligno
  Confianza < 65%:             El modelo no está seguro, consultar
                               a un especialista

7. TRAINING DASHBOARD - SECCIÓN ENTRENAMIENTO
===============================================================================

  Propósito: Entrenar modelos clásicos de clasificación.

  Modelos disponibles:
  - XGBoost (clasificador basado en árboles)
  - Random Forest (clasificador ensemble de árboles)

  Parámetros configurables:
  - n_estimators: número de árboles (10-500)
  - max_depth: profundidad máxima (2-20)
  - learning_rate: tasa de aprendizaje (solo XGBoost)
  - subsample/colsample: muestreo (solo XGBoost)
  - min_samples_split: división mínima (solo RF)

  Resultados mostrados:
  - Tabla comparativa de métricas (Accuracy, Precision, Recall, F1, AUC)
  - Gráfico de barras comparativo
  - Curvas ROC superpuestas
  - Matriz de confusión por modelo
  - Importancia de características
  - Interpretación textual automática

  Botón "Guardar Modelo": guarda el modelo como .pkl en Google Drive.

===============================================================================
8. TRAINING DASHBOARD - SECCIÓN VALIDACIÓN CRUZADA

===============================================================================
9. TRAINING DASHBOARD - SECCIÓN HIPERPARÁMETROS
===============================================================================

  Propósito: Encontrar la mejor combinación de parámetros.

  XGBoost evalúa:
    n_estimators: [50, 100, 200]
    max_depth: [4, 6, 8]
    learning_rate: [0.01, 0.1, 0.3]
    subsample: [0.6, 0.8, 1.0]
    colsample_bytree: [0.6, 0.8, 1.0]

  Random Forest evalúa:
    n_estimators: [50, 100, 200]
    max_depth: [4, 6, 8, None]
    min_samples_split: [2, 5, 10]
    min_samples_leaf: [1, 2, 4]

  Resultados:
  - Mejores parámetros encontrados
  - Heatmaps de interacción entre parámetros
  - Interpretación textual

===============================================================================
10. TRAINING DASHBOARD - SECCIÓN PRUEBAS ESTADÍSTICAS
===============================================================================

  Propósito: Validar si las diferencias entre modelos son significativas.

  Pruebas:
  - McNemar: compara predicciones categóricas (χ²)
  - Wilcoxon Signed-Rank: compara probabilidades (no paramétrica)

  Interpretación:
  - p < 0.05: diferencias significativas entre modelos
  - p >= 0.05: no hay evidencia de diferencia significativa

  Resultados:
  - Tabla de comparaciones por pares
  - Mapa de calor de p-valores
  - Interpretación general automática

13. SOLUCIÓN DE PROBLEMAS
===============================================================================

  Problema                              | Solución
  --------------------------------------|---------------------------------------
  "El servicio no está disponible"      | Esperar y recargar la página. Verificar
                                         que el backend esté corriendo.
  "Archivo no soportado"                | Convertir la imagen a JPG o PNG.
  "Archivo excede el límite"            | Reducir tamaño a menos de 5MB.
  Confianza baja (<65%)                 | El modelo no está seguro. Consultar
                                         a un especialista.
  Error de conexión a Drive             | Montar Google Drive en Colab o
                                         verificar ruta en config.py.
  No aparecen datos en EDA              | Verificar que Drive esté montado con
                                         la estructura correcta de carpetas.
  El entrenamiento no inicia            | Verificar que data.csv exista en
                                         BreastCancer_Tabular/.
  No se genera reporte                  | Verificar Results/ tenga permisos
                                         de escritura.

14. MÉTRICAS DE EVALUACIÓN
===============================================================================

  Accuracy:  Proporción de predicciones correctas sobre el total.
  Precision: De los clasificados como MALIGNANT, cuántos realmente
             lo son. (Importante para evitar falsos positivos)
  Recall:    De los MALIGNANT reales, cuántos fueron detectados.
             (Importante para evitar falsos negativos)
  F1-Score:  Media armónica de precisión y recall. Balance entre ambos.
  AUC-ROC:   Capacidad de discriminación entre clases.
             1.0 = perfecto, 0.5 = aleatorio.

  Interpretación de valores:
    0.90 - 1.00: Excelente
    0.80 - 0.89: Bueno
    0.70 - 0.79: Aceptable
    < 0.70:      Pobre (revisar modelo)
